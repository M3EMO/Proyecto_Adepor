## Anti-bot safeguards (heredadas + endurecidas)

- Sleep 1.5-3.5s entre calls (random jitter)
- Pausa 60s cada 50 calls
- User-Agent rotation (4 strings, Windows/Mac/Linux Chrome 130-131)
- 403 → abort total + log `aborted_reason='403_blocked'`
- 429 → backoff 30s + retry transparente
- Pause 32H entre sesiones para "cooldown" anti-detection

## Implementación

### Script principal
`scripts/scrape_sofa_backfill_historico.py`

```bash
# Manual
py scripts/scrape_sofa_backfill_historico.py --cap 1500
py scripts/scrape_sofa_backfill_historico.py --liga Argentina  # solo una liga
py scripts/scrape_sofa_backfill_historico.py --dry-run          # contar pendientes
```

Args:
- `--cap N` (default 1500): max calls SOFA por sesión
- `--liga LIGA`: filtrar a una liga
- `--fecha-min YYYY-MM-DD` (default 2022-01-01)
- `--fecha-max YYYY-MM-DD` (default 2026-12-31)
- `--dry-run`: solo contar, no scrapear

### Wrapper PowerShell (autogenerated)
`scripts/schedule_sofa_backfill_run.ps1`

Hace:
1. cd al proyecto
2. Ejecuta py con cap 1500 + tee logs a `scrape_sofa_backfill_logs/sofa_backfill_<ts>.log`
3. **Self-reschedules**: al terminar, crea task ONCE para +32H exacto

## Riesgos operativos

1. **403 bot detection**: si SOFA bloquea, task continúa intentando cada 32H pero falla todas. **Mitigación**: monitorear logs primeras 5 sesiones; si 3+ consecutivas con 403, suspender task manualmente y revisar UA/proxy.

2. **PC apagada**: task no corre si máquina apagada/dormida. **Mitigación**: schtasks tiene flag para ejecutar al despertar (`/RU` con power options) — agregable si se vuelve issue.

3. **DB lock**: si pipeline diario corre simultáneamente con backfill, posible SQLite WAL conflict. **Mitigación**: SQLite WAL mode + retry en errores temporales (no implementado, agregable si se observa).

4. **Calibración drift**: 48 días son largos. SHADOW filters pueden haber acumulado N≥80 antes de completar backfill → trigger PROPOSAL temprano. **OK**: re-validar con N grande post-backfill confirma o invalida.
